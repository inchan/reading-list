## Excerpt

기존 파일 탐색보다 3배 빠르게 파일을 검색 가능한 도두가 개발되었습니다.
무려 claude code 개발자 @boris_cherny 의 shoutout 을 받았어요. X 에서 나온 이야기를 봐서는 claude code 의 파일멘션(@) 도 이 엔진을 사용하도록 바뀐거 같아요.

코드를 분석해서 어떻게 동작하는지 파악해보았습니다.

이건 “새로운 검색 엔진을 처음부터 만든 프로젝트”가 아니라, pi의 기존 find/grep 기능을 Rust 기반 FFF 엔진으로 교체하는 얇은 extension이에요.

성능의 핵심은 pi-fff 자체가 아니라, 그 아래에 있는 @ff-labs/fff-node 엔진에서 나옵니다.

코드 규모는 매우 작고, 실제 핵심 로직은 src/index.ts 하나에 거의 다 들어있어요.

https://github.com/SamuelLHuber/pi-fff
